## 이상과 현실

### 프로세스의 변경
- 개인 의지에 기반한 구현의 생명력 한계
  - 만약 유지보수 인원이 바뀌거나 했을 때, 서비스의 접근성 수준을 올리거나 유지할 수 있을 것인가에 대한 고민.
  - 조직적인 의지와 에너지가 필요.
- 개발 프로세스에 포함되어야 지속적 개발운영이 가능함
  - 기획 -> 디자인 -> 개발 -> QA(1) -> QA(2)접근성 -> 운영
